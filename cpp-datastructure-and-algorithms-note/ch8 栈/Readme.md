ch8 栈

栈（Stack）是一种**后进先出（LIFO）**的受限线性表，只允许在栈顶进行插入（push）和删除（pop）操作。

问：栈和队列跟线性表有什么关系？
答：栈和队列在逻辑结构上都是线性表，但它们限制了插入和删除的位置，因此称为“受限线性表”，本质上是线性表的特殊形式。

后进先出（LIFO） 的线性结构




问：栈和队列替代数组或普通线性表的意义是什么？
答：不主要是为了更快的存取，而是为了符合问题的逻辑顺序。

栈：保证“后进先出”（LIFO），方便处理嵌套、回退等问题，例如函数调用栈、括号匹配、DFS。

队列：保证“先进先出”（FIFO），方便顺序处理任务，例如任务调度、BFS、打印队列。

问：为什么不用普通数组直接做？
答：普通数组或线性表可以随机访问，但难以自然表达“先进先出”或“后进先出”的逻辑，程序设计更复杂、易出错。


Stack（栈）操作时间复杂度
1️⃣ Array 实现（顺序栈）

| 操作         | 时间复杂度        | 原因                |
| ---------- | ------------ | ----------------- |
| push       | **O(1)**（均摊） | 直接在栈顶插入（扩容时例外）    |
| pop        | **O(1)**     | 仅移动 `top`         |
| top / peek | **O(1)**     | 直接访问 `array[top]` |
| isEmpty    | **O(1)**     | 判断 `top == -1`    |

2️⃣ LinkedList 实现（链栈）
| 操作         | 时间复杂度    | 原因        |
| ---------- | -------- | --------- |
| push       | **O(1)** | 头插法       |
| pop        | **O(1)** | 删除头结点     |
| top / peek | **O(1)** | 访问头结点     |
| isEmpty    | **O(1)** | 判断头指针是否为空 |
