ch10 跳表和散列

字典 dictionary


1. 散列表 (Hash Table)
定义： 散列表是一种通过**散列函数（Hash Function）**将特定键（Key）映射到内存中一个位置（Bucket/Slot）以实现快速访问的数据结构。
核心逻辑： 它建立了一种从“键空间”到“地址空间”的直接映射关系。
数学表达： 存储位置 $f(key)$。其中 $f$ 为散列函数。
物理存储： 通常基于数组。
冲突处理： 当两个不同的键算出相同的地址时（哈希冲突），通常采用“拉链法”（在冲突处挂链表）或“开放寻址法”（找下一个空位）来解决。

2. 跳表 (Skip List)
定义： 跳表是一种随机化的数据结构，它在有序链表的基础上通过建立多级并行索引，从而实现对数据的对数级（$O(\log n)$）检索、插入和删除。
核心逻辑： 空间换时间。通过在每个节点中存储多个前向指针，形成多条“快车道”。
物理存储： 基于链表。
层级特征： * 底层（Level 0）： 包含所有元素的有序单链表。
索引层（Level 1..n）： 每一层都是下一层的子集，层数越高，节点密度越低。
随机性： 一个节点属于哪几层由随机算法（如抛硬币）决定，这保证了结构的近似平衡，避开了红黑树复杂的旋转操作。
