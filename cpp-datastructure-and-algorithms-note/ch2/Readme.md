空间复杂度 = 程序在运行过程中额外占用的内存大小（随输入规模变化的部分）
但由于空间复杂度基本依靠编译器有关。很难计算，所以一般忽略不计
组成
指令空间
存放编译后的二进制机器码。在算法分析中，通常视为固定部分 $c$，忽略不计。
数据空间
全局变量、常量；动态数组 (new/vector) 存放在堆中。这是 S_P 的主要来源之一。
环境栈空间
每一次函数调用，栈里要保存：返回地址 形式参数 局部变量 寄存器状态等
存放函数调用的上下文。这是递归空间复杂度的核心来源
递归函数空间大小依赖于
形式参数所需要的空间
递归深度
递归分成两部分c+Sp（实力特征）Space=c+SP​
固定部分 c 是很小的开销 通常忽略）
编译后的机器指令（代码段）不管输入多大，程序代码大小不变
可变部分 Sp (实例特征)
 实力特征包括输入输出的数量或相关数大小



时间复杂度 = 程序运行所需要的时间
编译时间与实例特征无关
两种方法：
1. 找出关键操作确定执行时间，考虑最好和最坏
2. 确定程序总步数
先确定实例特征
确定一步指的是多少次运算/代码
可以设定一个统计步数的变量，每当一条语句执行一次，就++1
或者列出一张表格(s/e - 频率 - 相乘总步数)
